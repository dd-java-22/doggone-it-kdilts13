# Project Summary

## Overview

**Doggone It** is a mobile application that allows users to identify dog breeds from images using on-device machine learning. Users can analyze images, review predicted breeds, and save scans with additional information such as breed facts, notes, and favorites.

The application integrates local data persistence, device features (camera and gallery), and external API data to provide a complete end-to-end user experience.

---

## Current Functionality

The application currently supports the following features:

### Image Analysis

* Capture images using the device camera
* Select images from the device gallery
* Run on-device machine learning inference to generate breed predictions
* Display ranked predictions based on confidence
* Allow the user to select a prediction for further use

### Scan Management

* Save analyzed scans locally using a Room (SQLite) database
* Store associated predictions with each scan
* Persist user-selected breed choice

### Scan Details

* Display the original image and selected breed
* Retrieve and display breed facts from the Dog API
* Toggle between:

    * **Facts view** (API-provided information)
    * **Notes view** (user-entered text)
* Allow users to create and edit notes
* Allow users to mark scans as favorites

### Gallery Features

* Display saved scans in a grid layout using RecyclerView
* Support sorting of scans
* Support filtering to show only favorite scans
* Dynamically adjust column count based on user settings

### Settings

* Configure default sort option for saved scans
* Configure sort direction
* Configure gallery column count
* Persist settings using SharedPreferences

---

## Known Limitations

While the application is fully functional, there are a few inherent limitations:

* **Model accuracy** – Breed predictions are generated by a pre-trained machine learning model and may not always be correct.
* **Network dependency for breed facts** – Breed information requires an active internet connection and a valid Dog API key.
* **Local-only data storage** – All scans, notes, and favorites are stored locally on the device and are not synchronized across devices.
* **Device-dependent behavior** – Camera and gallery behavior may vary slightly depending on device and Android version.

These limitations are expected given the scope of the project and do not prevent normal usage.

---

## Missing or Deferred Features

The following features were considered but not implemented in this milestone:

* Cloud synchronization of saved scans and notes
* Advanced search functionality within saved scans
* Editing or replacing images for an existing scan
* Multi-user account management or remote authentication

These features were deferred to keep the project scope manageable.

## Cosmetic Improvements

The application is functionally complete, but additional UI polish could improve the overall user experience:

* Improved spacing and layout consistency across screens
* Enhanced styling for cards, buttons, and typography
* More refined empty states (e.g., no saved scans)
* Additional visual feedback for user actions (loading indicators, transitions)
* Minor alignment and visual consistency improvements across fragments

---

## Stretch Goals

Future enhancements that could expand the application include:

* Improving or fine-tuning the machine learning model for better accuracy
* Adding cloud storage or account-based synchronization
* Expanding breed information with additional APIs or richer datasets
* Implementing advanced filtering and search capabilities in the gallery
* Adding image editing or cropping before analysis
* Enhancing UI/UX with animations and more modern design patterns

---

## Summary

The application successfully implements all core requirements for the milestone, including:

* Multi-screen navigation
* Local database persistence
* Dynamic, data-driven UI
* Integration with device features (camera and gallery)
* Use of external services (Dog API)
* User-configurable settings

The app provides a complete and functional workflow from image capture through analysis, storage, and review, with opportunities for future enhancements.
